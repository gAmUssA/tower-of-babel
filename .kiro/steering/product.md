# Product Overview

## Tower of Babel to Babel Fish Demo

A demonstration project showcasing the evolution from chaotic polyglot Kafka architecture to Schema Registry-managed data contracts.

## Purpose

Educational demo illustrating how Schema Registry solves cross-language serialization challenges in distributed systems. Designed for 20-minute live presentations.

## Core Scenarios

1. **Tower of Babel** - Demonstrates serialization chaos across Java, Python, and Node.js
2. **Babel Fish** - Shows Schema Registry solution with automatic code generation
3. **Safe Evolution** - Demonstrates backward/forward compatible schema changes
4. **Prevented Disasters** - Shows how Schema Registry blocks breaking changes

## Key Principles

- **Concept clarity over realism** - Simplified examples for educational purposes
- **Schema-first development** - Central Avro schemas drive code generation
- **Polyglot architecture** - Three languages (Java, Python, Node.js) to demonstrate cross-platform compatibility
- **Type safety** - Compile-time validation across all languages through generated code
